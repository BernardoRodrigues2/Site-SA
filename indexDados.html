Introdução:


Os bancos de dados são sistemas usados para organizar e armazenar informações de forma segura e rápida. Para isso, utilizamos os SGBDs, que permitem criar, acessar e gerenciar esses dados com eficiência. A modelagem e os diagramas, como DER e MER, ajudam a visualizar como as informações estão conectadas, facilitando o desenvolvimento de sistemas bem estruturados.


O que é um SGBD? Tipos e aplicações:


Um SGBD (Sistema Gerenciador de Banco de Dados) é um software que controla, organiza e permite acessar um banco de dados. Ele que faz a “ponte” entre você e os dados.
Tipos:
Relacional -> usa tabelas, linhas e colunas (MySQL, SQL Server, PostgreSQL).
Não relacional (NoSQL) -> usado para dados mais flexíveis, sem estrutura fixa (MongoDB, Redis, Cassandra).
Hierárquico -> dados organizados como uma árvore.
Orientado a objetos -> armazena objetos completos.
Aplicações:
Sites, apps, lojas virtuais, sistemas de empresas, cadastros, jogos online…

O que é um dicionário de dados?


É como um “manual” do banco de dados. Ele descreve tudo: nomes das tabelas, campos, tipos de dados, tamanhos e regras.
*Tipos de dados mais comuns:
*INT -> números inteiros
*FLOAT/DOUBLE -> números decimais
*VARCHAR -> textos curtos
*TEXT -> textos grandes
*DATE, TIME, DATETIME -> datas e horários
*BOOLEAN -> verdadeiro/falso
*BLOB -> arquivos binários (imagens, docs...)

O que é uma modelagem de banco de dados? E o DER/MER?


Modelagem é o planejamento de como os dados vão ser organizados antes de criar o banco. É tipo fazer a planta de uma casa antes de construir.
*DER (Diagrama Entidade-Relacionamento): mostra as entidades (tabelas) e como elas se relacionam (1×1, 1×N, N×N).
*MER (Modelo Entidade-Relacionamento): é a mesma ideia, só que normalmente representa o modelo lógico/conceitual com mais detalhes sobre atributos e relacionamentos.

Exemplo explicado de um diagrama DER:


Imagina um sistema de escola:
*Entidade Aluno
*Entidade Turma
Relacionamento:
Um aluno pode estar em uma turma → 1:N (uma turma tem vários alunos).
O DER mostraria:
*Retângulo “Aluno”
*Retângulo “Turma”
*Entre eles, um losango “Está matriculado”
*Cardinalidade: Turma (1) → Aluno (N)

Exemplo explicado de um diagrama MER:


O MER é parecido, só que você inclui mais atributos dentro das entidades.
Usando o mesmo exemplo:
Entidade Aluno
*id_aluno (PK)
*nome
*data_nascimento
Entidade Turma
*id_turma (PK)
*nome_turma
*ano
Relacionamento: “matriculado”
No MER você veria as entidades com seus atributos dentro e o relacionamento conectando.
